# ParkSight — Backend (FastAPI)

FastAPI API server for the ParkSight enforcement command center. See the [root README](../README.md) for the full project overview and setup instructions.

## Run

```bash
# From the backend/ directory:
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Interactive docs: **http://localhost:8000/docs**

## Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| `GET` | `/health` | Liveness check |
| `GET` | `/api/stats` | Header KPIs + recommended fleet size |
| `GET` | `/api/hotspots` | All hotspots — filterable by `violation`, `station`, `min_cii`, `limit` |
| `GET` | `/api/hotspots/{id}` | Single hotspot detail |
| `POST` | `/api/optimize` | Optimized patrol deployment + coverage vs baselines |
| `GET` | `/api/coverage-curve` | Coverage % vs fleet size + elbow (recommended fleet) |
| `POST` | `/api/route` | TSP patrol circuit (road-snapped via OSRM or haversine fallback) |
| `GET` | `/api/blindspots` | Enforcement blind spots ranked by blind-spot score |
| `GET` | `/api/temporal` | Day × hour heatmap (IST) + peak hours |
| `GET` | `/api/breakdown` | Violation-type and vehicle-type distributions |

## Architecture

- **`main.py`** — FastAPI app + all endpoints
- **`core.py`** — Coverage index (BallTree) + greedy max-coverage optimizer + TSP solver (nearest-neighbour + 2-opt) — pure numpy, no heavy deps at runtime
- **`data.py`** — Data loading (hotspot CSV + clustered violations), blind-spot algorithm, stats/temporal/breakdown (all `lru_cache`'d)
- **`schemas.py`** — Pydantic request/response models (the API contract)
- **`precompute.py`** — One-time offline step: enriches hotspots with OSM `road_class` + `lanes` → writes `data/hotspots_enriched.csv`. Requires `osmnx`; only needed if re-generating from a new `hotspot_summary.csv`.

## Data dependencies

The server reads two files at startup:

| File | Generated by |
|------|-------------|
| `../output/hotspot_summary.csv` | `analysis/parking_hotspot_analysis.py` |
| `../output/clustered_violations.csv` | `analysis/parking_hotspot_analysis.py` |
| `data/hotspots_enriched.csv` *(optional)* | `precompute.py` (adds road geometry) |

If `data/hotspots_enriched.csv` is absent, the server falls back to `hotspot_summary.csv` with `impact = raw CII`.
